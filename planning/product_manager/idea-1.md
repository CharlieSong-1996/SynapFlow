# SynapFlow 画布数据处理模块 - 核心机制与交互 MVP

## 1. 产品理念 (Concept)
一个极致简化的、由 AI 深度赋能的画布式数据流处理工具。
只存在两种基础实体：**数据节点 (Data Node)** 与 **处理器节点 (Processor Node)**。

### 1.1 实体定义
* **数据节点 (Data Node)**: 代表一个被固化的数据视图（如 SQL 表格或 Query View），内含该表/视图的完整 Meta 信息（如列名称、数据类型、列描述、样本数据预览）。**数据本身是被动存在的**。
* **处理器节点 (Processor Node)**: 代表一段主动处理逻辑（Script/程序）。它是动态的，内嵌了一个专属的 AI Agent 对话上下文和记忆。用户通过与 Agent 对话，由 Agent 根据上游输入数据节点的 Meta 信息，自动编写或编辑脚本。

## 2. 核心用户旅程 (User Journey - "The Aha Moment")

1. **准备原始数据 (Data Preparation)**
   - 用户在画布上生成初始的【数据节点 A】和【数据节点 B】（例如从两张不同的数据库表加载）。
   - 用户可以双击节点，在侧面板预览它的 Meta 信息（列名、备注、前10行数据）。

2. **定义处理逻辑 (Process Setup)**
   - 用户在画布空白处点击或拖拽生成一个【处理器节点 P】。
   - 用户将【数据节点 A】和【数据节点 B】的输出端口连线至【处理器节点 P】的输入端口。

3. **AI 编程协作 (AI Code Generation)**
   - 用户双击打开【处理器节点 P】的配置面板。该面板核心是一个 Chatbot/代码编辑器混合界面。
   - 用户在对话框中输入指令：*“将输入A和输入B通过 user_id 进行 Left Join，并只保留最近30天活跃的用户数据。”*
   - 内置 Agent 自动读取 A 和 B 的 Meta 信息，生成对应的 Python/SQL 脚本，回显在节点内的代码区。

4. **执行与输出呈现 (Execution & Output)**
   - 用户点击“执行 (Run)”。
   - 脚本执行成功后，【处理器节点 P】的输出端口自动衍生出一个全新的【被生成的数据节点 C】。
   - 用户点击【节点 C】，立刻看到处理好之后的清洗聚合数据及更新后的 Meta 信息。
